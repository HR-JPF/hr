# هندسة الأوامر (Prompt Engineering) لدراسة وتصميم وكيل البايثون المحلي

**الهدف من هذا الملف:** 
هذا النص (Prompt) مُعد مسبقاً لتقوم بنسخه ولصقه في أي منصة ذكاء اصطناعي (مثل ChatGPT, Claude, Gemini, وغيرها) لكي يقوم بدراسة فكرة "الوكيل المحلي" (Local Python Worker) وإعطائك خطة هندسية وتصميمية متكاملة لبرمجته.

---

### 📋 انسخ النص الموجود بالأسفل وأرسله لوكلاء الذكاء الاصطناعي:

```text
أنت خبير متمرس في هندسة البرمجيات، تصميم النظم (System Architecture)، ودمج تقنيات الذكاء الاصطناعي. 

نحن نقوم ببناء منصة توظيف ذكية (ATS) تعتمد على الواجهة الأمامية React.js وقاعدة البيانات السحابية Supabase. 
لتقليل تكاليف استهلاك واجهات برمجة التطبيقات (API Tokens) السحابية لعمليات معالجة السير الذاتية، ولضمان استمرارية العمل (Fallback) عند تعطل مفاتيح الـ API، قررنا تصميم وبناء "وكيل ذكاء اصطناعي محلي" (Local AI Worker) مبرمج بلغة Python، سيعمل على أجهزة الشركة المحلية أو خادم افتراضي منخفض التكلفة (VPS).

**المهام الأساسية المخطط لها لوكيل البايثون (Python Worker):**
1. مراقبة قاعدة بيانات Supabase بشكل مستمر لاكتشاف أي سير ذاتية (CVs) جديدة تم رفعها من قبل المرشحين.
2. تحميل السيرة الذاتية (سواء كانت PDF أو Word أو صور) وتحويلها إلى نص.
3. معالجة النص محلياً باستخدام نماذج مفتوحة المصدر عبر برنامج مثل (Ollama).
4. استخراج البيانات المنظمة (الاسم، الخبرات، الشهادات، المهارات) وإرسالها ككائن JSON وحفظها مباشرة في قاعدة بيانات Supabase.
5. العمل كخيار طوارئ (Fallback Mechanism): إذا فشلت المنصة في تقييم إجابات المرشحين عبر الـ APIs السحابية (بسبب استنفاد الرصيد أو الحظر)، تقوم المنصة بتحويل أمر التقييم إلى هذا الوكيل المحلي ليقوم بالتقييم وإرجاع النتيجة.

**المطلوب منك كخبير:**
1. دراسة وتحليل هذه البنية المعمارية وتقييم مدى فاعليتها.
2. تقديم تصميم هيكلي (Architecture Design) مفصل لبرنامج البايثون. كيف سيتواصل مع Supabase؟ هل نستخدم Webhooks مع FastAPI، أم Realtime Subscriptions، أم دورة تحقق (Polling)؟ ولماذا؟
3. تحديد أفضل المكتبات (Python Libraries) وأدوات التشغيل التي تقترحها (مثل langchain, celery, pdfplumber، وما إلى ذلك).
4. ما هي الآلية الأفضل لضمان استقرار البرنامج محلياً وإعادة تشغيله تلقائياً في حال حدوث خطأ أو انقطاع في الاتصال أو الكهرباء؟
5. **أفكار بديلة ومقترحات إضافية:** هل لديك أي أفكار أخرى أو بدائل هندسية لتنفيذ هذه العملية بطريقة أكثر كفاءة، أسرع، أو أقل تعقيداً؟ أرجو طرح كل الحلول الإبداعية الممكنة التي قد تغيب عن ذهننا.

أريد الإجابة مفصلة، واضحة، ومنظمة، وتأخذ بعين الاعتبار أمان البيانات والكفاءة في استهلاك موارد الجهاز المحلي.
```
